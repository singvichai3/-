# คู่มืออัปเดตโปรแกรม รับเล่มรถ ตรอ.

เอกสารนี้ใช้สำหรับอ้างอิงเวลาออกเวอร์ชันใหม่ของโปรแกรม เพื่อให้ลูกค้าอัปเดตได้โดยไม่ต้องลบโปรแกรมแล้วลงใหม่เอง

## ไฟล์สำคัญ

- ตัวติดตั้งล่าสุด: `dist-1.0.4-update/รับเล่มรถ ตรอ Setup 1.0.4.exe`
- ไฟล์ตัวอย่าง manifest: `update-manifest.sample.json`
- จุดตั้งค่า URL อัปเดต: หน้า `ตั้งค่า` ภายในโปรแกรม

## ระบบอัปเดตทำงานอย่างไร

โปรแกรมจะอ่านไฟล์ JSON จาก URL ที่เราตั้งไว้

ไฟล์ JSON ต้องมีข้อมูลอย่างน้อย 2 ค่า:

- `version` = เวอร์ชันล่าสุด
- `url` = ลิงก์ตรงไปยังไฟล์ `setup.exe`

เมื่อผู้ใช้กด `ตรวจสอบอัปเดต`

1. โปรแกรมจะโหลดไฟล์ manifest
2. เปรียบเทียบเวอร์ชันปัจจุบันกับเวอร์ชันใหม่
3. ถ้ามีเวอร์ชันใหม่ จะถามผู้ใช้ว่าต้องการอัปเดตหรือไม่
4. ถ้าตกลง โปรแกรมจะดาวน์โหลด `setup.exe`
5. โปรแกรมจะเปิด installer ตัวใหม่ให้ติดตั้งทับของเดิม

## ขั้นตอนออกเวอร์ชันใหม่

### 1. แก้โปรแกรมให้เสร็จ

แก้โค้ดและทดสอบให้เรียบร้อยก่อน

### 2. เปลี่ยนเวอร์ชันใน `package.json`

ตัวอย่าง:

```json
"version": "1.0.5"
```

ควรเปลี่ยนใน `package-lock.json` ให้ตรงกันด้วย

### 3. build ตัวติดตั้งใหม่

ใช้คำสั่ง:

```powershell
npm run build
```

หรือถ้าต้องการแยก output ใหม่ชั่วคราว:

```powershell
.\node_modules\.bin\electron-builder.cmd --win --x64 --config.directories.output=dist-1.0.5
```

### 4. หาไฟล์ setup.exe ที่ build เสร็จ

ตัวอย่าง:

```text
dist-1.0.5\รับเล่มรถ ตรอ Setup 1.0.5.exe
```

### 5. อัปโหลดไฟล์ setup.exe ไปยัง server

ต้องเป็น URL ที่เข้าถึงได้จากเครื่องลูกค้า เช่น:

```text
https://your-server.example.com/downloads/รับเล่มรถ ตรอ Setup 1.0.5.exe
```

หมายเหตุ:

- ควรใช้ `https`
- ต้องเป็นลิงก์ดาวน์โหลดตรง ไม่ใช่หน้า preview
- ชื่อไฟล์มีภาษาไทยได้ แต่ URL ควรทดสอบจริงก่อน

### 6. สร้างหรือแก้ไฟล์ manifest

ใช้ `update-manifest.sample.json` เป็นต้นแบบ

ตัวอย่าง:

```json
{
  "version": "1.0.5",
  "url": "https://your-server.example.com/downloads/%E0%B8%A3%E0%B8%B1%E0%B8%9A%E0%B9%80%E0%B8%A5%E0%B9%88%E0%B8%A1%E0%B8%A3%E0%B8%96%20%E0%B8%95%E0%B8%A3%E0%B8%AD%20Setup%201.0.5.exe",
  "publishedAt": "2026-04-15T12:00:00+07:00",
  "notes": "- ปรับปรุงระบบ\n- แก้ข้อผิดพลาด\n- เพิ่มความเสถียร"
}
```

### 7. อัปโหลดไฟล์ manifest ไปยัง server

ตัวอย่าง:

```text
https://your-server.example.com/update.json
```

## วิธีตั้งค่าในโปรแกรม

ไปที่หน้า `ตั้งค่า`

กรอก URL ของไฟล์ manifest ลงในช่อง:

- `URL อัปเดตโปรแกรม`

จากนั้นกด:

- `บันทึกการตั้งค่า`

และเมื่อต้องการลองระบบอัปเดตให้กด:

- `ตรวจสอบอัปเดต`

## วิธีทดสอบก่อนส่งลูกค้า

### ทดสอบแบบพื้นฐาน

1. ติดตั้งโปรแกรมเวอร์ชันเก่า
2. ตั้งค่า URL manifest
3. เปลี่ยน `version` ใน manifest ให้สูงกว่าเวอร์ชันที่ติดตั้ง
4. กด `ตรวจสอบอัปเดต`
5. ตรวจว่าโปรแกรมดาวน์โหลด installer และเปิดตัวติดตั้งใหม่ได้

### สิ่งที่ควรตรวจ

- โปรแกรมต้องไม่ลบข้อมูลลูกค้า
- โปรแกรมต้องดาวน์โหลดไฟล์ได้จริง
- URL ของ `setup.exe` ต้องดาวน์โหลดตรง
- เวอร์ชันใน manifest ต้องมากกว่าเวอร์ชันที่ติดตั้งอยู่

## ปัญหาที่เจอบ่อย

### 1. กดตรวจสอบอัปเดตแล้วบอกว่าเป็นเวอร์ชันล่าสุด

สาเหตุที่เป็นไปได้:

- `version` ใน manifest เท่ากับเวอร์ชันปัจจุบัน
- ลืมแก้เลขเวอร์ชันใน manifest

### 2. โหลด manifest ไม่ได้

สาเหตุที่เป็นไปได้:

- URL ผิด
- server ปิดอยู่
- URL ไม่ใช่ `http` หรือ `https`
- ไฟล์ JSON ไม่ถูก format

### 3. ดาวน์โหลด installer ไม่ได้

สาเหตุที่เป็นไปได้:

- URL ของ `setup.exe` ผิด
- URL เป็นลิงก์ preview ไม่ใช่ลิงก์ไฟล์จริง
- server ป้องกันการดาวน์โหลดตรง

### 4. เปิด installer แล้วไม่อัปเดต

สาเหตุที่เป็นไปได้:

- ใช้ไฟล์เก่า
- เวอร์ชันใหม่ไม่ได้ build จริง
- ชื่อไฟล์ถูกแต่ข้างในยังเป็น build เก่า

## แนวทางการใช้งานจริง

ถ้าเริ่มใช้งานกับลูกค้า แนะนำให้ทำแบบนี้ทุกครั้ง:

1. แก้โค้ด
2. bump version
3. build setup.exe ใหม่
4. อัปโหลด setup.exe
5. อัปเดต `update.json`
6. ทดสอบจากเครื่องลูกค้า 1 รอบ
7. ค่อยปล่อยให้ลูกค้าทั้งหมดใช้งาน

## ข้อแนะนำเพิ่มเติม

- เก็บไฟล์ setup.exe เก่าย้อนหลังไว้เสมอ
- อย่าใช้ชื่อไฟล์ซ้ำกับคนละเวอร์ชัน
- ถ้ามีการแก้ฐานข้อมูล ควรทดสอบกับข้อมูลจริงสำรองก่อน
- ถ้าจะใช้งานกับลูกค้าหลายเครื่อง ควรมีที่เก็บไฟล์กลาง เช่น web server หรือ cloud storage ที่ให้ลิงก์ตรงได้

## สรุปสั้นที่สุด

เวลาออกอัปเดตใหม่ ให้ทำ 4 อย่าง:

1. เปลี่ยนเวอร์ชัน
2. build `setup.exe`
3. อัปโหลด `setup.exe`
4. แก้ `update.json`

หลังจากนั้นลูกค้าก็สามารถกด `ตรวจสอบอัปเดต` จากในโปรแกรมได้เลย
