# Checklist ปล่อยอัปเดตโปรแกรม

ใช้ลิสต์นี้ทุกครั้งก่อนปล่อยเวอร์ชันใหม่

## ก่อน Build

1. ยืนยันว่าแก้โค้ดและทดสอบจบแล้ว
2. เปลี่ยนเลขเวอร์ชันใน `package.json`
3. เปลี่ยนเลขเวอร์ชันใน `package-lock.json`
4. ตรวจว่า `build/icon.ico` ถูกต้อง
5. ตรวจว่า URL ปลายทางที่จะใช้ปล่อยไฟล์พร้อมใช้งาน

## ตอน Build

1. ปิดโปรแกรมที่เปิดค้างทั้งหมด
2. รันคำสั่ง build
3. ตรวจว่าได้ `Setup.exe` เวอร์ชันใหม่จริง
4. ตรวจว่าไฟล์ที่ได้ไม่ใช่ build เก่า

## หลัง Build

1. อัปโหลด `Setup.exe` ขึ้น server
2. แก้ `update.json` ให้ตรงกับเวอร์ชันล่าสุด
3. ตรวจว่า `url` โหลดไฟล์ได้จริง
4. ตรวจว่า `version` ใน `update.json` ตรงกับไฟล์ที่ปล่อย
5. ตรวจว่า `notes` อธิบายสิ่งที่เปลี่ยนชัดเจน

## ทดสอบก่อนส่งลูกค้า

1. ติดตั้งจากไฟล์ `Setup.exe` ใหม่
2. เปิดโปรแกรมแล้วลองเช็กอัปเดต
3. ทดสอบติดตั้งทับของเดิม 1 รอบ
4. ตรวจว่าข้อมูลลูกค้าไม่หาย
5. ตรวจว่าเปิดโปรแกรมซ้ำไม่ได้

## ก่อนปล่อยจริง

1. เก็บไฟล์ setup เวอร์ชันเก่าไว้สำรอง
2. ยืนยันว่าใช้ไฟล์ล่าสุดจริง ไม่ใช่ build เก่า
3. ส่งลิงก์หรือไฟล์ที่ถูกต้องให้ลูกค้า
4. จดบันทึกว่าเวอร์ชันนี้ปล่อยวันไหน
